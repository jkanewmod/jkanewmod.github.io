NewJK

NewJK is our optional improved custom executable that replaces jamp.exe, which launches when you start JA. It has many crash fixes and stability improvements that are not possible to address in Newmod. While NewJK is not required to use Newmod, we highly recommend it for improving JA's stability.

Newmod users should only use the vanilla executable or NewJK. Other projects (OpenJK, JAMME, etc.) are not recommended for use with Newmod, as we are not their maintainers and cannot guarantee their consistency with our vision for Newmod.

Non-Newmod users are fully encouraged to use NewJK. Newmod and NewJK are independent projects; neither requires the other. The benefits of NewJK can be enjoyed on any client mod, not just Newmod.

Installation

NewJK can be downloaded from the download page.

Compared to your current setup, if you switched to NewJK, you would gain these features:

  • Many crash fixes and stability improvements.
  • The new cvar r_noBorder can be enabled to run in borderless windowed mode.
  • The new cvar r_centerWindow can be enabled to automatically position non-fullscreen JA in the center of your screen.
  • The new cvar r_gammaShaders can be enabled to make r_gamma only affect the JA window, which can be useful for non-fullscreen JA or multi-monitor setups.
  • r_mode can be set to -2 for automatic screen resolution detection.
  • Maximum cvars raised from 1224 to 8192.
  • Multiple JA windows can be run simultaneously without crashing.
  • Working Alt + Tab as well as minimize command.
  • Reduced minimum input lag when moving mouse or pressing any key.
  • Fixed lag when taking your first screenshot of the session.
  • screenshot command without argument uses timestamp for filename instead of base JA numbering system.
  • screenshot command no longer requires quotation marks for custom filenames containing spaces.
  • Support for orange and grey text colors with ^8 and ^9.
  • Extended valid range of cl_timeNudge to allow values between -1000 and 1000.
  • Extended valid range of cl_maxPackets to allow values between -20 and 1000.
  • Enter being able to auto-complete commands from the main menus, instead of just from in-game.
  • Fixed duplicated projectile effects bug (e.g. seeing seven repeater shots when you only clicked once).
  • Fixed some siege textures not producing proper footstep effects.
  • Fixed echo command not properly preserving colors between spaces.
  • Fixed crash when starting local server with widescreen resolution.
  • Fixed crash when looking at Ragnos NPC.
  • Fixed broken rendering with invalid primitives.
  • Fixed rendering of transparent shadowed text.
  • Server list no longer considers colors and non-alphanumeric characters for sorting by name.
  • demo command no longer requires quotation marks for filenames containing spaces.
  • Fixed the "jittery text" bug that makes small text appear ugly.
  • Malicious servers are prevented from forcing invalid/troll cvars on your client.
  • Removed ugly white box flash from some explosion effects.
  • CD check bypass allows playing non-Steam JA without the disc inserted.
  • Enabled scrolling in console with mouse wheel.
  • Unrecognized console commands are no longer sent as chat messages.
  • Ugly "RECORDING XYZ.dm_26" text is removed.
  • Fixed connection hijack exploit on malicious servers.
  • Fixed pressing Esc during demo playback causing you to exit to the main menu.
  • Restored vanilla behavior of pressing Enter to auto-complete commands, including in the main menus.
  • Restored vanilla behavior of color codes being hidden while typing.
  • The cfg that automatically gets read/written has been restored to the vanilla filename jampconfig.cfg.
  • Vanilla-style portablility is used by default. So instead of using Documents/My Games/NewJK for game files (or whatever fs_homepath is set to), classic single-folder behavior is used. Combined with the jampconfig.cfg change, this allows you to easily swap executables without needing to mess with folders or cfgs. If you prefer using OpenJK's default fs_homepath behavior, you can still do that by adding +set fs_portable 0 as an argument when running NewJK.
  • JA now defaults to running in full screen at your native resolution, just like any modern PC game (r_fullscreen 1 and r_mode -2).
  • Projectile duplication/stretching has been fixed (e.g. seeing seven repeater shots when you fired once).
  • Extended valid range of cl_timeNudge to allow values between -1000 and 1000.
  • Allows loading of patched map files (e.g. maps/mp/mapname.bsp.patched) so as to use clientside map patches without causing autodownload to start.
  • screenshot command no longer requires quotation marks for custom filenames containing spaces.
  • Ugly "RECORDING XYZ.dm_26" text is removed by default.
  • Server list no longer considers colors and non-alphanumeric characters for sorting by name.
  • Fixed pressing Esc during demo playback causing you to exit to the main menu.
  • demo command no longer requires quotation marks for filenames containing spaces.
  • Removed ugly white box flash from some explosion effects.
