Download the latest version of the NewJK executable for Windows (March 31, 2023).

While NewJK is not required to use Newmod, we highly recommend it.

NewJK is our optional improved custom executable that replaces the vanilla JKA executable (jamp.exe). As a custom executable, it is capable of implementing many improvements that are not possible with simple PK3 mods (like Newmod). It can improve FPS, has many crash fixes and stability improvements, and adds extra features like raw mouse input. For more information, see the NewJK page.

If you are also already using Newmod, make sure to update that, too.

To install NewJK manually, put jamp.exe, rd-vanilla_x86.dll, SDL2.dll, steam_api.dll, and steam_appid.txt in the folder where your existing jamp.exe is located (this is usually the /GameData/ folder). We recommend backing up your existing jamp.exe first.

Release notes for 1.10.0:




    New announcer features

    • Added a new Announcer Voice (cg_announcerVoice) cvar, which allows you to use custom announcer voices.
      • Newmod includes kyle, desann, jan, and luke.
      • Newmod also includes shortened versions of these voices: kyle_short, desann_short, jan_short, and luke_short.
      • Newmod automatically switches to short-version voices when several sounds occur in quick succession.
    • Added a new Announcer Dropped (cg_announcerDropped) cvar, which enables "flag dropped" sounds.
    • Added a new Announcer Channels (cg_announcerChannels) cvar, which allows ally/enemy/neutral announcer sounds to play in separate audio channels.
    • Added a new Announcer Cooldown (cg_announcerCooldown) cvar, which adjusts how rapidly the announcer begins new sounds.
    • All new announcer features are disabled by default.

    Other sound features

    • Added a new Frag Sounds (cg_fragSounds) cvar, which plays a sound when you frag an enemy.
    • Added a new Announcer Volume (s_volumeAnnouncer) cvar, which controls the volume of the announcer (this was previously controlled by the more general s_volume). On first boot, this will be automatically be set to whatever you have s_volume set to.
    • Added a new Master Volume (s_volumeMaster) cvar, which all other volume settings are multiplied by.

    Player-specific forced models

    • You can now persistently forcemodel a specific player (across multiple sessions) by using the new forceModel subcommand of the Player Tracker (nm_playerTracker) command.
    • Renamed the old nm_forceModel command to nm_tempForceModel.

    Miscellaneous

    • Added a new inactiveopacity= parameter to the Strafe Helper (cg_strafeHelper).
    • Implemented proper clientside prediction for some tweaks on base_enhanced servers.
    • Fixed a couple of minor bugs.
