To install Newmod manually, put newmod.pk3 and assetsNewmod.pk3 in your /base/ folder. Remove any conflicting mods, including Smod-related files.
To install NewJK manually, put jamp.exe, rd-vanilla_x86.dll, and SDL2.dll in the folder where your existing jamp.exe is located (this is usually the /GameData/ folder). We recommend backing up your existing jamp.exe first.

Release notes for 1.1.0:

  • Disruptor shots and trip mine lasers are no longer prevented from rendering if the source is behind you.
  • Added support for multiple master servers. Instead of "Internet" source for servers, you can now choose from any servers specified in the cvars sv_master1 through sv_master5. Master 1 is already set to Ravensoft; Newmod sets master 5 to JKHub's master server for you. Ravensoft's master server has already gone down periodically, and some day it will go down permanently. Now you can use another master server when that happens.
  • WeaponCFGs now work when following people from spectator mode. In addition, whenever you go into "free spectator" mode, generic.cfg and the new freespec.cfg will execute (if they exist). This includes when connecting, loading a new map, joining the spectators, pressing Space while following someone, temporary Siege dead mode, etc. The generic CFG should suffice for most users; you shouldn't really need to use freespec.cfg. This is mainly intended so that when you go into free spec mode you don't get stuck with the settings from whichever weapon CFG you last used.
  • Fixed some bugs and alignment issues relating to some HUD elements.
  • Hid some annoying messages from appearing in-game, such "Recording to xxx.dm_26"
  • Fixed some minor bugs with the Newmod menu.
  • Miscellaneous bug fixes and improvements.
Remember that users running both Newmod and NewJK should update both.
