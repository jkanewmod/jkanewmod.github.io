Download the latest version of the NewJK executable for Windows (January 22, 2023).

While NewJK is not required to use Newmod, we highly recommend it.

NewJK is our optional improved custom executable that replaces the vanilla JKA executable (jamp.exe). As a custom executable, it is capable of implementing many improvements that are not possible with simple PK3 mods (like Newmod). It can improve FPS, has many crash fixes and stability improvements, and adds extra features like raw mouse input. For more information, see the NewJK page.

If you are also already using Newmod, make sure to update that, too.

To install NewJK manually, put jamp.exe, rd-vanilla_x86.dll, SDL2.dll, steam_api.dll, and steam_appid.txt in the folder where your existing jamp.exe is located (this is usually the /GameData/ folder). We recommend backing up your existing jamp.exe first.

The 1.9.2 minor update fixes a startup issue on the vanilla executable, adds the new Siege Timer Style (cg_siegeTimerStyle) cvar, and fixes a couple of minor bugs.

Release notes for 1.9.0:


    Saber combat & movement

    • Added a new Disable Saber Moves (cg_disableMoves) cvar, which allows you to selectively disable special saber moves (staff kick, butterfly, etc.) on supporting servers.
    • Implemented proper clientside prediction for the "superman jump" fix on base_enhanced servers.

    See force alignment of teammates

    • Added a new Team Force Alignment Display (cg_drawTeamForce) cvar, which displays light/dark side force alignment of teammates in the team overlay and carrier display on supporting servers.

    Improved FPS

    • Added a new 4 bitflag to the FPS Saver (cl_fpsSaver) cvar, which removes decorative map effects such as flames. This can significantly improve FPS on maps with many decorative effects, such as ctf_dantooine.
    • Improved the 2 bitflag of the FPS Saver (cl_fpsSaver) cvar to reduce FPS loss caused by rocket trails and force push/pull.

    Weapon changing

    • Added a new Low Ammo Switch (cg_lowAmmoSwitch) cvar, which you can use to prevent automatically switching weapons if you attempt to fire with low ammo. By default, for example, if you attempt to alternate fire golan with 10 ammo, you are forced to switch to another weapon. However, because you could still primary fire golan with 10 ammo, you may not want to switch weapons. This cvar allows you to toggle this automatic switch.
    • Added a new Weapon Cycle Skip (cg_weaponCycleSkip) cvar, which prevents equipping specific weapons while cycling through weapons. This setting is most useful if you cycle through weapons but also use binds to equip some weapons. For example, many players cycle through guns but use the 1 key for saber/melee. Skipping saber also helps avoid undesired "unholstering" animation lock.
    • Added a version of the Weapon Group (nm_weaponGroup) command that cycles in the reverse direction: nm_weaponGroupPrev.

    Miscellaneous

    • Implemented proper clientside prediction for several small fixes and improvements on base_enhanced servers.
    • Flag possession icons no longer fade to black while you are falling into a pit.
    • Flag possession icons for dropped allied flags now include a faint X underneath the ↓ arrow for better visual consistency, and to better distinguish them from dropped enemy flag icons.
    • Fixed forced names not being used in "Waiting for <name> to unAFK" messages on base_enhanced servers.
    • Fixed some small bugs relating to ghosting dead players in Siege.
    • Other minor fixes and improvements.
